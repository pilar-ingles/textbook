Язык CSS - начало

В веб-разработке язык CSS (Cascade Style Sheets, каскадные таблицы стилей) позволяет придать странице конкретный вид. Он состоит из правил, которые применяются к элементам (селектам)страницы. В качестве селекта выступает любой HTML-элемент, для которого определяются правила форматирования (стили).

Способы подключения стилей

  1. Для использования у конкретного тега.

    Стиль задается непосредственно в теге с помощью атрибута style. Применяется только к данному тегу.

    Правило пишется в следующем виде: <тег style="свойство: значение; свойство: значение;">

    Примеры:

    <p style="color: red;"> Содержание </p> - Текст абзаца выводим на экран в красном цвете.

    <p style="margin-top: -15px; text-indent: 50px;"> Содержание </p> - Поднимаем вверх абзац на 15px и сдвигаем вправо на 50px.

    <h3 style="margin-top: -10px;margin-bottom: -15px; text-align: center"> Содержание </h3> - Убираем отступы перед и после заголовка и центрируем его.

  2. Для использования на одной странице сайта, но в нескольких местах.

    Стиль задается в заголовке страницы (head) с помощью тега style.

    Правило пишется в следующем виде: <style> тег {свойство: значение;} </style>

    Пример: <style> p {color: red;} </style>. - Текст всех абзацев выводим на экран в красном цвете.

  3. Для использовании на всех страницах сайта

    Cоздается файл с расширением .css, в котором прописываются все необходимые для создания сайта стили.

    Такой файл может храниться в одном каталоге с файл index.html, а может в отдельном каталоге.

    Файл стилей подключается в заголовке страницы (head)помощью тега link: <link rel="stylesheet" href="Путь_к_файлу_стилей">.

    Пример: Чтобы текст всех абзацев на всех страницах сайта вывести на экран в красном цвете, создадим в папке проекта (в одном каталоге с файлом index.html) файл main.css следующего содержания: p { color: red; }.

    А в разделе head пишем <link rel="stylesheet" href="main.css" >

Правила стилизации в файле стилей CSS

  1. В CSS-код могут вставляться специальные директивы, которые начинаются с символа @.
    • @font-face{свойства шрифта} – подключение сторонних шрифтов;
    • @import(url файла) – подключение сторонних стилей;
    • @media(параметры) – указание медиа-носителя.
  2. Правила в файле CSS располагаются в той последовательности, в которой они должны выполняться. Всегда выполняется первое правило, которое нашлось для данного селекта. Следует заметить, что правила работают для любого уровня вложенности указанных в них элементов. Правило одинаково сработает и для первого и для второго уровней вложенности.
  3. Для создания правила сначала указывается селект, к которому будет применяться стиль. Затем идет фигурная скобка, правило в виде пары "свойство: значение", точка с запятой, следующее правило и т.д. Заканчивается фигурной скобкой. Вид правила стиля: селект {свойство: значение; свойство: значение;}
  4. В файлах CSS можно писать комментарии, текст которых располагается между символами /* ...*/. Вид комментария: /* Это стили */

Что может выступать в качестве селекта в правиле?

  • Имя тега.

    Если необходимо оформить одним стилем одинаковые теги, то указывается имя тега, к которому должен примениться стиль. Стиль применится ко всем тегам с таким именем.

    Пример: p {color: red;} - Все параграфы будут оформлены красным цветом.

  • Идентификатор (id у HTML-тега).

    По правилам написания HTML-кода у любого тега может быть идентификатор, но с одним условием - он не может повторяться. При создании правила перед именем нужного идентификатора ставится символ"#".

    Пример: Для смены цвета у параграфа с id="uniq" создадим правило #uniq {color: red;}, а в теле кода HTML пишем <p id="unig"> Содержание </p>

  • Класс.

    По правилам написания HTML-кода любому тегу может быть определен класс. При создании правила по классу перед именем класса ставится точка ".".

    Для примера окрасим элемент с классом class-name в красный цвет. Сначала создаем для данного класса правило в файле стилей: .class-name {color: red;}, а в теле кода HTML указываем этот класс у нужного тега: <p class="class-name"> красный цвет </p>

    Если у элемента несколько классов, то к нему будут применены стили, объявленные для всех этих классов. Стили применяются в порядке их появления в файле CSS, поэтому если для разных классов объявляются одинаковые стили, то они будут перезаписывать друг друга.

    Если нужно отстилизовать элемент с одним классом, находящийся внутри элемента с другим классом, то имена классов в селекторе нужно писать через пробел, означающий вложенность. Можно указывать не только имена классов, но и любые другие элементы.

    Например: правило p .btn { color: red;} применится ко всем элементам с классом btn, находящимся внутри абзаца.

    Если нужно применить стили к элементам, принадлежащим одновременно нескольким классам, то в коде HTML имена этих классов перечисляются друг за другом через пробел, а в правиле перед именем каждого класса ставится точка без пробела.

    Например: правило .one.two.three.four { color: red;}. будет применено к элементу <div class="one two three four">, но не будет применено к <div class="one two">.

  • Атрибуты тега.
    1. Атрибуты указываются в квадратных скобках.

      Например: Правило [href] {color: red;} применится ко всем элементам, имеющим атрибут href.

    2. Можно указать не только имя атрибута, но и его значение. Для этого после атрибута ставится знак "=" и указывается значение.

      Например: [href = "/flower.jpg"] {color: red;}

    3. Можно указать только начало значения атрибута, а не точное значение. В этом случае ставится символ "^".

      Например: [href ^= "/flower"] {color: red;}

    4. Можно указать строку, которой заканчивается значение атрибута. В этом случае ставится символ "$"

      Например: [href $= "flower.jpg"] {color: red;}

    5. Можно указать данные из середины значения атрибута. В этом случае поставит символ "*"

      Например, [href *= "flower"] {color: red;}

    6. Атрибут может иметь несколько значений, которые разделены пробелами, а необходимо выбрать с каким-то конкретным. Используется символ "~".

      Например: [class ~= "another-class"] {color: red;}

      В HTML-коде: <div class="class-name another-class"> содержание </div>

    7. В HTML-коде слова в названиях классов и идентификаторов разделяются дефисами. В CSS есть возможность выбирать элементы по словам, встречающимся в таких составных именах классов или идентификаторов. Для этого используется символ "|".

      Например: в HTML-коде есть элемент с классом example-class-name.

      Правило [class |= "example"] {color: red; } будет применено ко всем элементам, название классов которых начинается на слово "example", после которого идет дефис.

Универсальные правила

  1. Символ "*" приводит к выбору всех тегов на странице. * {color: red;} Применяется для задания глобальных настроек, которые нужно применить к каждому элементу (например, поведение блочной модели).
  2. Символ "+" позволяет выбирать соседей в объектной модели документа. Пусть нам нужно применить стили только к абзацу (тег p), который идет непосредственно после элемента с идентификатором top. Для этого используется селектор со знаком "+", позволяющий выбрать элемент относительно его ближайшего соседа сверху: #top + p {color: red; }
  3. Символ "~" задает правило смежных блоков. Изменим наш селектор на следующий: #top ~ p {color: red;}. В этом случае стили применились ко всем тегам p, которые находятся на одном уровне с элементом, имеющим идентификатор top. Даже если эти теги будут разделены другими, то они всё равно попадут в выборку и к ним применятся стили.
  4. Правило первого уровня. Пусть нам нужно применить стили к тегам li, относящимся только к внешнему списку с классом list, а теги li, находящиеся внутри вложенного списка с классом inner-list, затрагивать не нужно. Если использовать уже знакомый нам вложенный, то будут выбраны все теги li, находящиеся внутри элемента с классом list. .list li { border: 1px dashed #ccc;} Нам же надо отобрать только теги li, относящиеся к тегу ol с классом list, поэтому нужно применить селектор со знаком "<", который позволяет выбрать только непосредственных потомков: .list < li {border: 1px dashed #ccc;} Теперь стили применились только к тегам li, являющимся непосредственными потомками элемента с классом list.

Псевдоклассы (псевдоселекторы)

Рассмотрим группу правил, которая называется псевдоклассы или псевдоселекторы (также можно называть их селекторами состояний).

    Названия псевдоселекторов в правиле пишутся после двоеточия
  • :hover - Псевдоселектор указывает на то, что правило должно быть применено к элементу только при наведении на него мышки.

    Например, мы хотим, чтобы при наведении курсора мыши на кнопку её заливка стала синей. .button:hover {background: blue;}

  • :active - Псевдоселектор позволяет применять стили к элементу только после щелчка мышью на этом элементе.
  • Для одного элемента можно указать несколько псевдоселекторов.

    Например, кнопка в начальном состоянии имеет оранжевую заливку, но применение следующих стилей приведёт к тому, что заливка кнопки будет синей при наведении на неё курсора мыши и станет зелёной после щелчка на ней мышью.

    .button {background: orange;}

    .button:hover {background: blue;}

    .button:active {background: green;}

    Подобные псевдоклассы можно применять к любым элементам на странице.
  • :target - Псевдосект позволяет, например, подсветить блок к которому произведен переход по якорной ссылке. p:target {background: green;}

    Стили из этого правила (зелёный цвет фона) применятся к абзацу (тег p) только в случае перехода к нему по якорной ссылке. Они будут применяться до тех пор, пока из адресной строки не будет убрана якорная ссылка, а страница будет перезагружена.

  • :visited - Псевдоселектор позволяет задавать стиль для ссылок, по которым уже осуществлялся переход.a:visited { color: green; } Если теперь щелкнуть мышью на ссылку, то откроется другой сайт. Вернувшись на свою страницу и перезагрузив её, мы увидим, что текст ссылки отображается зелёным цветом.
  • :focus - Псевдоселектор применяется только совместно с элементами форм. Стили, заданные в правиле с псевдоселектором :focus, будут применены при получении фокуса элементом формы.

    Например, следующее правило изменяет цвет фона элемента формы с классом text-input при получении фокуса этим элементом. .text-input:focus {background: yellow; }

  • :disabled - Псевдоселектор позволяет стилизовать отключенные элементы в форме.

    Например, отключим с помощью атрибута disabled текстовое поле в нашей форме. <input type="text" class="text-input" disabled> Определим стиль для отключенных элементов с классом text-input: .text-input:disabled {background: magenta; }

  • :checked - Псевдоселектор позволяет применить стили для выбранного чекбокса (элемент <input type="checkbox">) или радиокнопки (элемент <input type="radio">).

    Однако напрямую стилизовать чекбоксы и радиокнопки не получится, так как внешний вид этих элементов зависит от используемого браузера и операционной системы. Но мы можем скомбинировать селекторы для стилизации, например, слова, расположенного рядом с чекбоксом. Добавим в нашу форму тег span, в содержимом напишем слово "Да". .check:checked + .span {color: red; } Здесь селектор выделяет элемент с классом span, который расположен после элемента с классом check, находящегося в выбранном состоянии (checked). Теперь после выбора чекбокса слово "Да" становится красным. Именно по этому принципу чаще всего на реальных проектах выполняют стилизацию чекбоксов и радиокнопок в формах.

Также последний пример является иллюстрацией того, что селекторы и псевдоселекторы разных типов можно комбинировать друг с другом. Однако переусердствовать с этим не стоит, так как изменять в дальнейшем подобные цепочки из селекторов разных типов может быть тяжело.

Псевдоклассы порядка

    Псевдоклассы по порядку элементов внутри родителя.

  • :first-child - Правило будет применено к первому элементу в списке: li:first-child {color: red;} Данное правило будет выбирать элементы во всех структурах, где li является потомком первого уровня.
  • :last-child - Псевдокласс выбирает последние дочерние элементы. li:last-child {color: red;}
  • :nth-child(№) - Псевдокласс позволяет выбирать элементы с произвольными порядковыми номерами (нумерация начинается с единицы). Например, следующее правило будет применено к пятому элементу в списке: li:nth-child(5) {color: red;}
  • :nth-child(even|odd) - Псевдокласс для выбора четных (even) или нечетных (odd) элементов li:nth-child(even) {color: red;}
  • :nth-child(an+b), где a и b – некоторые числа, а n=0, 1,2, …. - Псевдокласс для выбора дочерних элементов с номерами, удовлетворяющими этой формуле.

    Например, следующее правило будет применено к третьему, пятому и седьмому элементам (2*0+3=3, 2*1+3=5, 2*2+3=7): li:nth-child(2n + 3) {color: red;}

  • псевдокласс :not - можно назвать исключающим првилом. Этот псевдокласс позволяет убрать из выборки элементы, удовлетворяющие правилу, переданному в :not. Например, добавим в index.html четвёртому тегу li класс active и напишем правило, которое будет применено ко всем элементам li, кроме элемента с классом active. li:not(.active) {color: red;}

Псевдоэлементы :before и :after

Внутри почти всех тегов HTML (за исключением тегов форм и тегов встраиваемого контента (изображения, аудио, видео и т. п.)) есть ещё два дополнительных элемента :before и :after, которые позволяют добавлять стилизованное содержимое до и после элемента, к которому они применяются. Добавим в main.css два правила для определения содержимого псевдоэлементов :before и :after с помощью свойства content.

.block:before {content: "до"; }

.block:after {content: "после";}

Кроме непосредственно контента (текста) в правила для псевдоэлементов можно добавлять любые нужные CSS-свойства.

Например: .block:before {content: "до"; color: red;} .block:after {content: "после"; color: blue;}

Чаще всего псевдоэлементы :before и :after используются в декоративных целях, чтобы привести верстку в соответствие с макетом, не добавляя при этом лишние сущности в HTML-файлы. При этом свойство content может оставаться пустым, однако следует запомнить, что если вообще не указывать свойство content, то псевдоэлемент просто не будет создан.

Приоритет правил

К одному и тому же элементу могут применяться стили из разных правил. При этом надо помнить, что правила, написанные в файле стилей ниже, имеют больший приоритет, чем правила с такими же селекторами, указанные выше.

Однако, при анализе в CSS правил для определённого элемента учитываются правила приоритета. У каждой группы есть определённый вес и чем больше этот вес, тем большим приоритетом он будет обладать. Общее правило можно сформулировать так: "Чем точнее правило, тем он весомей и приоритетней".

    Разные типы имеют разные веса:

  • Вес по идентификатору равен 100.
  • Вес атрибутов, вес по классам и псевдоселекторы равен 10.
  • Вес тегов и селекторы псевдоэлементов равен 1.
  • Вес универсальных правил равен 0.
  • Если вес совпадает, то приоритет отдается тому правилу, которое определено позднее (расположено в файле стилей ниже).
  • При комбинировании вес складывается. Поэтому, если скомбинировать по идентификатору и по классу, то вес станет равным 110 и приоритет будет выше приоритета по идентификатору. Отметим, что это правило приоритета только в случае, когда мы пытаемся изменить одно и то же свойство у одного и того же блока в разных правилах. Если же в правиле устанавливается какое-либо свойство, которого нет в других правилах, то оно будет применено без учета приоритета селекторов.
  • В CSS существует возможность искусственно повысить приоритет определённого правила, указав в конце него ключевое слово !important. Например, повысим приоритет у селектора по идентификатору в правиле, устанавливающем синий цвет шрифта: #uniq-id {color: blue !important; } Однако применять директиву !important при написании собственного кода CSS не рекомендуется, так как её использование может сделать файл стилей сложным для понимания и отладки. Вместо этого лучше разобраться в коде и найти реальную проблему, которая скорее всего связана со слишком сложным построением.
  • Инлайновые стили (стили, написанные непосредственно в HTML-разметке элемента с помощью атрибута style) имеют больший приоритет, чем стили из файла стилей, за исключением стиля с директивой !important.
