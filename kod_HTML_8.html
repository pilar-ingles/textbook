HTML - Функциональные теги

Тег Формат Атрибуты Описание
script - Парный <script type="тип" src="URL"> Содержание</script> type - Указывает тип MIME для определенного языка (type="тип"). Атрибут type не чувствителен к регистру и принимает следующие значения: text/javascript-для JavaScript,text/vbscript-для VBScript. Значение по умолчанию text/javascript

src - Импортирует файл со скриптами из внешнего файла. Как только внешние инструкции полностью загружаются, браузер обрабатывает их, словно они были внедрены в текущий документ. Файлы со скриптами JavaScript обычно имеют расширение js и могут размещаться в любом доступном месте.(src="URL")

 Предназначен для описания скриптов, может содержать ссылку на программу или ее текст на определенном языке. Скрипты могут располагаться во внешнем файле и связываться с любым HTML-документом. может располагаться в заголовке или теле HTML-документа в неограниченном количестве.
link <link Атрибуты > charset - Указывает кодировку документа на который ссылается тег link (charset="кодировка")

href - Путь к файлу, на который делается ссылка(href="URL")

media - Определяет устройство, для которого следует применять стилевое оформление. Это позволяет сделать разный стиль для отображения документа на экране монитора и при его печати. Допускается писать несколько значений через запятую.(media="all-все|braille-уст-ва для слепых|handheld-Наладонники, смартфоны|print-принт|screen-монитор|speech-речевые синтезаторы|projection-проектор|tty-портативные уст-ва|tv-ТВ")

rel - Определяет отношения между текущим документом и файлом, на который делается ссылка. (rel="тип"). Значения:stylesheet-подключаемый файл хранит таблицу стилей (CSS). alternate-альтернативный тип, например, для указания ссылки на файл в формате XML для описания ленты новостей, анонсов статей.

sizes - Указывает размер иконок для визуального отображения. Сама иконка может применяться браузером для отображения в адресной строке, при сохранении в избранное, а также поисковыми системами для придания наглядности результатам поиска (sizes="<ширина>X<высота> <ширина2>X<высота2> | any"). Ширина и высота в пикселях без указания единиц (например, 10), между нимилатинская буква x в любом регитсре. Можно задавать размеры нескольких иконок через пробел, "any"-масштабируется в любой размер.

type - Сообщает браузеру, какой MIME-тип данных используется для внешнего документа.Применяется для того, чтобы указать, что подключаемый файл содержит CSS. Пишется в любом регистре. Для подключаемых таблиц связанных стилей применяется тип text/css.(type="MIME-тип")

 Устанавливает связь с внешним документом вроде файла со стилями или со шрифтами. Размещается всегда внутри контейнера head.
meta <meta Атрибуты > charset - Указывает кодировку документа.(charset="кодировка", значения "UTF-8"|"RU")

name="название-метаданных" content="значение" - name и content — это атрибуты тега meta, которые работают в паре. Они передают браузерам и поисковым системам дополнительную информацию о странице; name — указывает, какого типа информация передаётся (например, описание страницы, ключевые слова, автор и так далее),content — содержит саму эту информацию.

name="author" content="Иван Петров"

- Указывает автора

name="generator" content="WordPress 6.0" - указывает программу или систему, которая использовалась для создания страницы (например, WordPress, Joomla, FrontPage).

name="description" content="Описание страницы для SEO" - краткое описание страницы

name="keywords" content="HTML, метатеги, SEO, веб-разработка" - список ключевых слов

name="robots" content="index, follow" - управление индексацией и сканированием. Значения атрибута content: index — разрешить индексацию страницы; noindex — запретить индексацию; follow — разрешить переход по ссылкам на странице; nofollow — запретить переход по ссылкам; none — сокращение для noindex, nofollow. По умолчанию: index, follow стоит по умолчанию

name="viewport" content="width=device-width, initial-scale=1.0" - Указывает браузеру, как масштабировать и отображать страницу на разных устройствах, особенно на смартфонах и планшетах. (Viewport — это область, которую видит пользователь на экране, когда заходит на страницу сайта с любого устройства). Он позволяет указать, какая область просмотра необходима для страницы . viewport необходим для корректной мобильной адаптации и влияет на SEO: поисковые системы отдают приоритет сайтам, оптимизированным для смартфонов. Если этого тега нет или он указан с ошибкой, позиции сайта в выдаче могут пострадать. Атрибут width=device-width указывает, что ширина страницы должна соответствовать ширине экрана устройства. Значение initial-scale=1.0 устанавливает масштаб по умолчанию — 100%, без увеличения или уменьшения.

Некоторые основные параметры viewport (content):

  • width определяет ширину области просмотра. Чаще всего используется значение device-width, что означает «использовать ширину экрана текущего устройства».
  • initial-scale задаёт начальный масштаб страницы. Значение 1 означает, что страница будет отображена без масштабирования.
  • maximum-scale и minimum-scale определяют, как сильно пользователь может увеличивать или уменьшать масштаб.
  • user-scalable имеет значения «yes» или «no» и определяет, может ли пользователь масштабировать страницу.

name="theme-color" content="#4285f4" - цвет интерфейса браузера. Цвет задаётся в шестнадцатеричном формате (HEX): #4285F4 — синий; #FF0000 — красный.

http-equiv="refresh" content="30" - Автоматически обновить страницу. Браузер обновит страницу через 30 секунд.

http-equiv="refresh" content="5;url=https://example.com/new-page.html" - Настроить временное перенаправление. Браузер перейдёт на указанный адрес через пять секунд. Для постоянных редиректов лучше использовать серверные настройки, — так их корректно обработают поисковые системы.

http-equiv="Content-Security-Policy" content="default-src 'self'" - Защитить сайт от XSS (content security policy): Этот метатег помогает защитить сайт от XSS-атак, при которых злоумышленник пытается внедрить вредоносный код на сайт. Метатег запрещает загружать ресурсы со сторонних источников, если они не разрешены явно. Обычно такая политика задаётся на стороне сервера, но в простых случаях её можно прописать и через — при тестировании сайта или при локальной разработке.

Частые ошибки

  • Кодировка указана не первой в head. Браузер должен узнать о кодировке как можно раньше. Если кодировка указана после других тегов, текст может отображаться неправильно.
  • Есть ошибки во viewport. Для адаптивной вёрстки важно, чтобы тег был написан правильно. Даже небольшая ошибка может повлиять на отображение сайта на мобильных устройствах. Если пропущен width=device-width,то браузер не поймёт, что нужно подстроить ширину страницы под ширину экрана смартфона. В результате сайт может отображаться слишком узким или слишком широким, и пользователю придётся вручную уменьшать или увеличивать масштаб.
  • 3. Повторяются теги meta с одинаковыми name. На странице должен быть только один тег с name="description", "viewport", "robots" и другие, иначе браузеры и поисковые системы не поймут, какой из них использовать. Повторения могут вызвать конфликт или игнорирование тегов.
  • Описания пустые или слишком длинные. Слишком короткие или, наоборот, перегруженные описания могут ухудшить SEO и просто не будут учтены поисковыми системами. Если описание вообще отсутствует — в таком случае поисковик сам подставит какой-то фрагмент текста со страницы. Это не всегда будет удачно: может попасться неподходящий абзац или фраза без смысла. Если слишком длинное описание: Такой текст просто не влезет в сниппет — его обрежут поисковые системы, и пользователь не увидит главного. Описание должно укладываться в ~150 символов, содержать ключевые фразы (iPhone 15, цена, доставка, цвета), привлекательно выглядит в поисковой выдаче.

Определяет метатеги, которые используются для хранения информации предназначенной для браузеров и поисковых систем. Например, для получения описания сайта, ключевых слов и других данных. Разрешается использовать более чем один метатег, все они размещаются в head.
