Работа с бесплатными хостингами

1. Хостинг GitHub Pages https://github.com
GitHub— платформа в интернете, где программисты хранят и делятся своим кодом, где можно поместить общий проект, совместно управлять всеми изменениями, а в случае неудачи — быстро вернуться к исправной версии.

GitHub Pages — сервис платформы GitHub, который позволяет размещать статические веб-сайты прямо из репозитория GitHub. Он поддерживает файлы HTML, CSS и JavaScript, интегрирован с генератором статических сайтов Jekyll. Сервис позволяет разработчикам и техническим специалистам демонстрировать свои работы, делиться информацией, создавать и размещать личные проекты, портфолио, документацию и блоги. GitHub Pages подходит для разработки и поддержания актуальности веб-контента.

Преимущества использования GitHub

  • автоматическое обновление сайта при каждом изменении в репозитории;
  • возможность использовать шаблоны оформления и язык разметки Markdown;
  • поддержка плагинов — на сайт можно добавить комментарии, слайдеры, формы для авторизации.

    Способ1

    1. Открыть меню пользователя. Для этого в правом верхнем углу страницы нужно кликнуть по своей аватарке или иконке пользователя.
    2. Перейти в «Settings».
    3. Найти «Preferences» в разделе «User settings».
    4. Прокрутить страницу вниз, пока не появится раздел «Language». Здесь будет выпадающий список со всеми доступными языками, выбрать «Русский».
    5. Сохранить изменения, нажав кнопку «Save changes». После этого следует обновить страницу, чтобы изменения вступили в силу.

    Ограничения при использовании GitHub Pages:
    • Объём хранилища — каждый проект может занимать не более 1 ГБ свободного места.
    • Лимит на исходящий трафик — 100 ГБ в месяц. Если планируется размещать на GitHub Pages тяжёлые мультимедийные файлы, видео или аудио, то этот лимит может быстро исчерпаться.
    • Запрещено использовать GitHub Pages для коммерческих проектов, например площадок e-commerce. Также сайты GitHub Pages не должны использоваться для конфиденциальных транзакций, таких как отправка паролей или номеров кредитных карт.
    Порядок работы
    1. Регистрация на сайте GitHub. Для этого нужно зайти на сайт https://github.com и нажать кнопку Sign up «Регистрация». Находится в правом верхнем углу страницы.
    2. Создание аккаунта. На странице регистрации ввести адрес электронной почты, пароль и имя пользователя (login). Имя пользователя должно быть уникальным. Для регистрации на GitHub необходима электронная почта в зоне .com, например почта Google. При попытке зарегистрировать аккаунт на GitHub на .ru почту, после заполнения “Капчи” выйдет ошибка "Unable to verify your captcha response"("Не удалось проверить ваш ответ капчи"). Если почта выглядит как my-name@yandex.ru, то можно использовать альтернативный адрес: my-name@yandex.com без каких-либо дополнительных настроек. Все письма, отправленные на yandex.com автоматически будут перенаправлены на основную почту Яндекс. Такой вариант подходит для регистрации на GitHub. У mail.ru и других российскох почтовых сервисов .com двойника изначально нет.
    3. Подтверждение email. На указанный email придёт письмо со ссылкой для подтверждения регистрации. Надо перейти по ссылке, чтобы завершить регистрацию. После авторизации откроется главная страница аккаунта.
    4. Создание репозитория. Репозиторий на GitHub — это пространство для хранения и управления исходным кодом для заданного проекта. Он содержит все файлы, исходный код, документацию и другой контент, необходимый для проекта. Репозитории могут быть открытыми или закрытыми. Открытые (публичные) доступны всем в интернете, закрытые (приватные) — только авторизованным пользователям.
      • На главной странице аккаунта переходим на вкладку «Repositories»(Репозитарий).
      • Для создания репозитория надо нажать на «+» в правом верхнем углу сайта, выбрать пункт «New Repository» (Новый)
      • «Repository name» - Введем имя (название) создаваемого репозитория. Для личных сайтов (обычно для портфолио) — имя в формате .github.io
      • «Description» - Добавим краткое описание в поле (необязательно)
      • «Public/Private» — выбрать видимость репозитория (публичный или приватный)
      • «Initialize this repository with a README» - поставим галочку, если нужно создать файл README.md в корне репозитория.
      • Нажать на кнопку «Create repository»
    Ошибки при создании репозитория на GitHub

    Есть несколько причин, по которым не создаётся репозторий на GitHub:

    1. Неправильный уровень видимости. При создани репозитория надо выбрать правильный уровень видимости (публичный, приватный). При этом важно убедиться, что у пользователя есть права на создание приватных репозиториев, если выбран этот тип.
    2. Конфликт с игнорированием файлов. Если используется файл .gitignore, нужно убедиться, что в нём не указаны какие-либо системные или временные файлы, которые могут вызвать конфликт.
    3. Неправильная лицензия. Если при создании репозитория выбирается лицензия, нужно убедиться, что выбранная лицензия действительна и е вызывает ошибку.
    4. Ограничения аккаунта. Иногда учётные записи на GitHub могут иметь ограничения по созданию репозиториев. Стоит проверить свой аккунт на наличие каких-либо ограничений.
    5. Проблемы с кэшем браузера. Если при создании репозитория ранее возникала ошибка, стоит очистить кэш браузера и повторить попытку.
    6. Ошибка «REPOSITORY CREATION FAILED». Если при создании репозитория на GitHub появилась такая ошибка, то можно попробовать следующее:
      • Подождать и повторить попытку. Иногда GitHub может занимать несколько минут, чтобы полностью удалить репозиторий и освободить имя.
      • Проверить имя репозитория. Убедиться, что используемое имя ещё не занято и соответствует правилам именования на GitHub.
      • Проверить проблемы с браузером. Стоит очистить кэш и файлы cookie или использовать другой браузер, чтобы понять, сохраняется ли проблема.
      • Проверить статус GitHub. Стоит посмотреть на странице статуса, нет ли текущих инцидентов, которые могут влиять на создание репозитория.
      • Обратиться в поддержку GitHub. Специалисты предоставят более конкретное руководство с учётом учётной записи и предпринятых действий.
    Минимальные настройки для репозитория:
    • Для GitHub Codespaces. Если проекту требуется определённый уровень вычислительной мощности, можно настроить так, чтобы пользователи могли использовать только типы компьютеров, удовлетворяющие этим требованиям. Для этого нужно добавить параметры в файл devcontainer.json, например, свойство hostRequirements с указанными параметрами (количество ядер процессора, память, хранилище). После сохранения файла и фиксации изменений в требуемой ветви репозитория при создании codespace для этой ветви можно будет выбрать только те типы компьютеров, которые соответствуют укаанным ресурсам или превышают их.
    • Для репозитория дистрибутивов и контейнеров. В этом случае минимальные шаги по настройке включают определение имени репозитория, URL для удалённого хранилища, дистрибутива и хранилища Blob для хранения данных. Также для некоторых репозиториев, например, Git, минимальная настройка может включать установку и настройку SSH-сервера для безопасного доступа к серверу, инициализацию проекта на сервере и настройку файла конфигурации репозитория (config). Конкретные минимальные настройки зависят от используемого репозитория и его особенностей.
    Настройки репозитория находятся н странице репозитория, на вкладке «Настройки», во вложенной вкладке «Репозиторий». Они доступны владельцам или соавторам с правами администратора.
    • Название репозитория, веб-сайта и описание. Выполняется на вкладке «Настройки» во вложенной вкладке «Репозиторий». После внесения изменений надо нажать кнопку «Обновить».
    • Видимость репозитория. Определение приватности или публичности влияет на то, кто может видеть содержимое и историю измеений репозитория.
    • Передача прав собственности на репозиторий. Позволяет изменить владельца репозитория, что может быть полезно при передаче проекта другим разраотчикам или организациям.
    • Архивация репозитория. Позволяет сохранить текущее состояние репозитория в виде архива, который можно загрузить и распаковать в будущем.
    Создание папки в репозитории на GitHub

    В GitHub нельзя создать пустую папку. Сначала создается файл внутри репозитория. Название файла формируется латинскими буквами, английским языком и в нижнем регистре. Если в имени несколько слов, то используйте знак нижнее подчеркивание "_", либо тире "-".

    • Перейти в нужный репозиторий.
    • Нажать на кнопку «Добавить файл», затем на ссылку «Создать новый файл» в выпадающем меню. В результате откроется окно с описанием вариантов добавления файлов в репозиторий через консольные команды. (Также на этой странице есть ссылка "uploading an existing file" для загрузки файлов через графический интерфейс, которой мы и воспользуемся.)
    • Ввести имя директории, за которым следует символ косой черты (например, empty_folder/).
    • Ввести имя файла.
    • Нажать на кнопку «Сохранить изменения».
    • Написать сообщение коммита, выбрать ветку, в которую коммитить файлы, и снова нажать на кнопку «Сохранить изменения» для сохранения изменений.
    Заливка файлов на GitHub:
    • Перейти на главную страницу репозитория.
    • Над списком файлов выбрать выпадающее меню «Добавить файл» и нажать «Залить файлы».
    • Выбрать файлы для загрузки в открывшемся файловом браузере.
    • В поле «Сообщение коммита» ввести краткое описание вносимых изменений. Теперь для сохранения изменений в репозитории и публикации их на сервере нужно создать коммит. В описании коммита мы укажем сделанные изменения.
    • Нажать зелёную кнопку «Зафиксировать изменения» (Commit changes).
    Добавить репозиторий в сервис GitHub Pages.
    • Открыть вкладку Settings (Настройки).
    • Страницу с настройками прокрутить вниз до раздела GitHub Pages.
    • В поле "Source" выбрать источник файлов для GitHub Pages (в нашем случае это основная ветвь main нашего репозитория), указать корневой каталог (root) и нажать кнопку Save.
    • В разделе GitHub Pages появится сообщение, что проект опубликован и доступен в виде веб-страницы по адресу, содержащему имя вашего GitHub-аккаунта и название репозитория.
    • Для того, чтобы проект стал доступен в GitHub Pages, требуется некоторое время, поэтому если сразу перейти по указанной ссылке, то мы увидим сообщение о том, что страница не найдена.
    • Через несколько минут сайт будет опубликован и фон сообщения с адресом нашего проекта в разделе GitHub Pages станет зеленым.
    • По указанной ссылке открывается содержимое нашего репозитория.
    • Если страница долго не появляется и постоянно выходит сообщение об ошибке 404, то это может быть связано с тем, что браузер закешировал эту страницу. В этом случае попробуйте перезагрузить браузер.
    Внесение изменений в репозитарий в сервис GitHub Pages.
    • Перейти в проект и внесем изменения в любой файл.
    • Загрузить измененный файл в репозиторий на GitHub. Для этого в раскрывающемся списке "Add file" выбрать пункт "Upload files".
    • Откроется уже знакомая нам страница, на которую нужно перетащить файл и сделать описание нового коммита. Нажать кнопку "Commit changes".
    • На странице с содержимым репозитория мы увидим, что файл обновился.
    • При нажатии на сообщение коммита, можно увидеть, какие именно изменения были сделаны в файле.
    • Откроем страницу нашего проекта на GitHub Pages и убедимся, что изменения были применены и на странице отображается новое содержимое файла.
    Сделать репозиторий на GitHub публичным.
    1. Перейти к приватному репозиторию, который нужно изменить.
    2. Нажать на вкладку «Настройки» в правом верхнем углу страницы репозитория.
    3. Прокрутить страницу настроек вниз до раздела «Опасная зона» (Danger Zone).
    4. Нажать кнопку «Изменить видимость репозитория» и выбрать «Изменить на публичное».
    5. Во всплывающем окне подтвердить выбор кнопкой «Я хочу сделать этот репозиторий публичным».
    6. Нажать на кнопку «Я прочитал и понимаю эти эффекты».
    7. Нажать «Сделать этот репозиторий публичным». После того как репозиторий станет публичным, весь код и данные в нём будут доступны всем на GitHub.
    Пригласить друга в репозиторий GitHub.
    1. Перейти на главную страницу репозитория.
    2. Под названием репозитория нажать «Настройки» (Settings). Если вкладки «Настройки» нет, выбрать выпадающее меню, затем нажать «Настройки».
    3. В разделе «Доступ» на боковой панели нажать «Соавторы» (Collaborators). Раздел «Collaborators» в подразделе «Access» из бокового меню слева. Он посвящён взаимодействию пользователей внутри репозитория.
    4. Нажать «Добавить людей» (Add people). После нажатия откроется всплывающее окно с возможностью выбора имени пользователя.
    5. В поле поиска начать набирать имя человека, которого хотите пригласить, затем нажать на имя в списке совпадений. После нажатия на имя нужного пользователя название кнопки поменяется на «Add <имя пользователя> to this repository», и она станет активной.
    6. Нажать «Добавить имя в репозиторий» (Add NAME to REPOSITORY), чтобы отослать приглашение пользователю. Пользователь получит приглашение на электронную почту. После подтверждения он получит доступ к репозиторию.
    Ошибка при подключении пользователя к репозитарию.«Uploads are disabled. File uploads require push access to this repository» в GitHub.

    Для исправления ошибки надо включить push-доступ к репозиторию:

    1. Перейти в репозиторий на GitHub.
    2. Нажать на «Настройки» на панели инструментов.
    3. Выбрать «Соавторы и команды» и «Добавить людей».
    4. Ввести имя пользователя GitHub того, кого нужно добавить, предоставить ему доступ на запись и нажать «Добавить человека в этот репозиторий».
    5. Приглашённому человеку проверить электронную почту, найти приглашение и нажать «Просмотреть приглашение».
    6. Нажать «Принять приглашение».
    7. Появится страница с подтверждением, что у пользователя есть push-доступ.
    Что делать, если не можете зайти в аккаунт на GitHub. Если профиль не найден, и username is case-sensitive, можно попробовать следующее:
    • Проверить правильность ввода имени пользователя. Нужно убедиться, что введена верная комбинация букв и цифр, включая заглавные буквы и специальные символы.
    • Проверить видимость профиля. Например, в GitHub нужно проверить, не установлен ли профиль на «частный». Если да, следует нажать на кнопку «Редактировать профиль» и изменить видимость на «публичный».
    • Подождать после изменения имени пользователя. Если имя было изменено недавно, может потребоваться время, чтобы платформа обновила изменения.
    • Проверить статус платформы. Стоит посетить страницу со статусом, чтобы убедиться в отсутствии ongoing сбоев или технических проблем.
    • Обратиться в поддержку платформы. Если ничего из вышеперечисленного не помогло, можно нажать на кнопку «Помощь» в правом верхнем углу страницы и выбрать «Связаться со службой поддержки».
    Как получить id репозитория на GitHub? Чтобы получить ID репозитория на GitHub, можно открыть в браузере URL-адрес api.github.com. В ответе JSON будет поле id, которое и является идентификатором репозитория.
    Для работы с GitHub API можно сгенерировать личный токен доступа (PAT) в настройках учётной записи GitHub. Имея личный токен доступа, можно включить его в заголовки запросов API для аутентификации и выполнения действий от имени своей учётной записи. Для этого нужно:
    1. Перейти в настройки своей учётной записи GitHub, щёлкнув свой аватар профиля в правом верхнем углу и выбрав «Настройки».
    2. В левой боковой панели нажать «Настройки разработчика».
    3. Нажать «Личные токены доступа».
    4. Нажать кнопку «Создать новый токен».
    5. Указать описательное название токена и выбрать желаемые области (разрешения) на основе действий, которые нужно выполнить.
    6. Нажать «Создать токен».
    7. Скопировать сгенерированный токен и хранить его в безопасном месте.
    2. GitVerse — российский аналог GitHub https://gitverse.ru/home/
    GitVerse — российский аналог GitHub, платформа для совместной разработки и хостинга кода, созданная компанией «СберТех» (входит в «Сбер»). GitVerse создан для того, чтобы сохранить отечественные IT-проекты в случае блокировки GitHub в России. Некоторые особенности GitVerse:
    • Поддержка GigaCode. Это встроенный ИИ-программист, который подсказывает лучшие варианты для написания кода и исправляет допущенные ошибки.
    • Совместимость с более чем 15 языками программирования.
    • Возможность подключать различные плагины, например, нейросети и облачные сервисы.
    • Бесплатная платформа, плагины распространяются по подпискам.
    3. Git - локальная версия для установки ПК.
    Устанавливать Git на локальный ПК для полноценной работы с GitHub это необязательно, но позволяет вести проект на своём локальном компьютере. Для установки зайти на git-scm.com, выбрать установщик для нужной платформы и скачать его. Установить Git по инструкциям программы.
    1. Чтобы создать пустой репозиторий в Git, нужно:
      • Создать пустую директорию для будущего репозитория и перейти в неё: mkdir myproject cd myproject
      • Выполнить команду git init. В результате будет создан новый локальный пустой репозиторий. На экран будет выведено сообщение вида: «Initialized empty Git repository in /path/to/myproject/.git/».
      • В директории myproject появится скрытая папка .git. Её можно увидеть, выполнив команду ls-al.
      • Чтобы репозиторий стал менее пустым, нужно добавить файлы в репозиторий и создать первый коммит: git add . git commit -m "Мой первый коммит"
    2. Добавление проекта в локальный репозиторий:
      • Скопировать файлы своего проекта в каталог локального репозитория.
      • В терминале перейти в каталог репозитория с помощью команды cd
      • Выполнить следующие команды для добавления файлов проекта в репозиторий: git add . git commit -m "Initial commit"
    3. Ещё один способ — использовать терминал:
      • Создать пустой репозиторий на GitHub. Для этого нужно нажать кнопку «+» в верхнем углу и выбрать «Новый репозиторий».
      • На локальном компьютере открыть терминал и перейти в папку с проектом, который нужно загрузить.
      • Запустить в папке команду git init. Она инициализирует папку и организует её так, чтобы её можно было использовать как репозиторий.
      • Сказать Git, какие файлы добавить в репозиторий, запустив команду git add . Она добавит все файлы в текущей директории на стадию подготовки.
      • Запустить команду git commit -m «начальный коммит» для создания коммита с описанием «начальный коммит».
      • Запустить команду git branch -M main для установки основной ветки.
      • Подключить локальный репозиторий к репозиторию на GitHub, запустив команду git remote add origin и заменив URL-адрес URL, предоставленным на странице инструкций GitHub при создании репозитория.
      • Запустить команду git push -u origin main для отправки всех файлов на GitHub. github.blog
      • После этого файлы должны появиться на GitHub
      • Загрузить файлы. Для этого нужно зайти в нужный репозиторий, щёлкнуть на Add file и выбрать Upload files.
    4. Загрузка проекта на GitHub:
      • В терминале ввести следующую команду для загрузки изменений на GitHub: git push origin main
      • Если используется ветка «master» вместо «main», заменить «main» на «master» в команде. Для работы с GitHub на смартфоне можно использовать мобильное приложение сервиса. В нём, например, можно создать новую проблему, нажав на «+» в правом верхнем углу страницы, а доступ к репозиториям получить в разделе «Мои работы».
    5. Ещё один способ — использовать pull-запрос:
      1. Скачать репозиторий на локальный компьютер.
      2. Открыть его в Windows Explorer (Finder на Mac) и добавить файл, который нужно загрузить, в соответствующую папку.
      3. Вернуться в консоль или окно терминала и запустить команды: cd .\AutoMapper\ git checkout -b AddingFile git add . git commit -m «Adding a file» git push
      4. Вернуться к вилке репозитория в браузере. Если GitHub предлагает создать pull-запрос прямо на главной странице, нужно нажать «Создать pull-запрос».
      5. Владельцы исходного репозитория будут уведомлены, они рассмотрят изменение файла и решат, принимать его или нет.
    6. Клонирование репозитория на локальный компьютер:
      1. Установить Git на свой компьютер, если ещё не сделано.
      2. Открыть терминал (на Windows использовать Git Bash) и перейти в каталог, где нужно разместить свой проект.
      3. Ввести следующую команду, заменив и на свои данные: git clone https://github.com//.git
