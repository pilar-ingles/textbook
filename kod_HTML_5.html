img <img src="путь к изображению" alt="Описание картинки" width="" height="" align="">
Примеры указания пути
  1. путь к файлу в Интернете

    <img src="https://www.flylady.ru/images/butterfly.png" >

  2. путь к файлу на локальном ПК

    <img src="file:///C:\Users\ВЛ\Desktop\IMG-20.jpg">

  3. путь к файлу из проекта

    <img src="./picture/IMG-20190929-WA0000.jpg">
  • src–путь к изображению (обязательный элемент)
  • alt-описание изображения, которое отображается на странице в случае проблемы с кзагрузой изображения
  • width-ширина изображения в пикселях
  • height-высота изображения в пикселях
  • align-ориентация изображения относительно текста, может принимать значения left—слева, right—справа,center-в центре
  • loading-позволяет регулировать загрузку, может принимать значения: eager—немедленно, lazy—медленно(изображение загрузится тогда, когда пользователь дойдёт до него на странице).
  • decoding-рекомендации по декодированию изображения, для быстроты загрузки страница сайта (улучшает производительность и снижает время загрузки страниц). Значения: sync — синхронно для одновременного отображения с другим контентом; async — асинхронно, чтобы уменьшить задержку отображения другого контента; auto — без предпочтений (режим по умолчанию).
Вывод изображения. Если задается ширина, то высота подстраивается, сохраняя пропорции изображения, и наоборот. Но ести задаются оба атрибута, то изображение выводится, используя оба, не соблюдая пропорции.

audio

<audio controls> <source src="имя файла" type="тип файла" > Сообщение, если браузер не поддерживает формат аудио </audio>

Упрощенный синтаксис (без source) актуален, если все браузеры поддерживают указанный формат аудио:

<audio src="путь и имя файла" type="тип controls"> Сообщение, если браузер не поддерживает формат аудио.</audio>

Примеры использования

  1. Использование нескольких атрибутов одновременно

    <audio src="audio1.mp3" controls autoplay loop muted preload="auto"> Браузер не поддерживает формат. </audio>

  2. Аудиофайл находится в той же директории, что и HTML-файл

    <audio src="audio1.mp3" controls> Браузер не поддерживает формат.</audio>

  3. Аудиофайл находится на другом сервере

    <audio src="https://pict.com/audio1.mp3" controls>Браузер не поддерживает формат.</audio>

  4. Аудио с элементами управления. Можно воспроизводить, приостанавливать и регулировать громкость аудио.

    <audio controls> <source src="audio1.mp3" type="audio/mpeg"> Браузер не поддерживает формат.</audio>

  5. Автоматическое воспроизведение при загрузке страницы и повторяется после завершения (фоновая музыка).

    <audio src="audio1.mp3" controls autoplay loop> Браузер не поддерживает формат.</audio>

  6. Аудио без звука (для тестирования)

    <audio src="audio1.mp3" controls muted>Браузер не поддерживает формат.</audio>

  7. Несколько источников аудио, для обеспечения совместимости с разными браузерами. Браузер по порядку пытается воспроизвести форматы один за другим.

    <audio controls> <source src="audio1.mp3" type="audio/mpeg"> <source src="audio1.ogg" type="audio/ogg"> Браузер не поддерживает формат. </audio>

src-указывает путь к аудиофайлу. Аналогично img

type-тип файла:для MP3-audio/mpeg; для Ogg-audio/ogg; для Wav-audio/wav

controls-показывает элементы управления воспроизведением.

autoplay-автоматически воспроизводит аудио при загрузке страницы.

loop-повторяет аудио после завершения воспроизведения.

muted-воспроизводит аудио без звука.

preload-определяет загрузку аудио. Значения: none-воспроизводится пользователем; metadata-загружаются только метаданные аудиофайла (например, длительность). auto-Загружается полностью при загрузке страницы.

Встраивает аудиофайлы на веб-страницы

Форматы аудио: Для обеспечения поддержки на всех браузерах, лучше использовать несколько форматов аудио (например, MP3 и OGG).

Атрибута controls: Всегда давайте возможность пользователям управлять аудиоплеером.

Атрибут preload: Значение auto дает быстрый доступ к аудио, но увеличивает время загрузки страницы.
