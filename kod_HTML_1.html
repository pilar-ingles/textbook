HTML - Теги для описания документа (голова документа -head)

Тег Формат Атрибуты Описание
link <link Атрибуты > Устанавливает связь с внешним документом вроде файла со стилями или со шрифтами. Размещается всегда внутри контейнера head.
<link charset="кодировка" > charset="кодировка".
    Значения кодировки
  • UTF-8
  • RU
 Указывает кодировку документа
<link href="URL"> href="URL" URL - Путь к файлу, на который делается ссылка
<link media="тип устройства"> media="тип устройства"
    Типы устройств:
  • "all" - все;
  • "braille" - устройства-ва для слепых;
  • "handheld" - Наладонники, смартфоны;
  • "print" - принтеры;
  • "screen" - мониторы;
  • "speech" - речевые синтезаторы;
  • "projection" - проекторы;
  • "tty" - портативные уст-ва;
  • "tv" - ТВ
 Определяет устройство, для которого следует применять стилевое оформление. Позволяет задать разный стиль для отображения на разных устройствах. Можно писать несколько значений через запятую.
<link rel="тип"> rel="тип".
    Значения типов:
  • "stylesheet"-подключаемый файл хранит таблицу стилей (CSS).
  • "alternate"-альтернативный тип, например, для указания ссылки на файл в формате XML для описания ленты новостей, анонсов статей.
 Определяет отношения между текущим документом и файлом, на который делается ссылка.
<link sizes="ШхВ"> sizes="ШхВ", где Ш-ширина, В-высота.
    Значения:
  • ШхВ - Ширина и высота указываются в пикселях без указания единиц (например, 10), между ними латинская буква x в любом региcтре.
  • any-масштабируется в любой размер.
  • sizes="шXв ш2Xв2" - Можно задавать размеры нескольких иконок через пробел
Указывает размер иконок для визуального отображения. Сама иконка может применяться браузером для отображения в адресной строке, при сохранении в избранное, а также поисковыми системами для придания наглядности результатам поиска.
<link type="MIME-тип"> type="MIME-тип"
    Значения "MIME-тип". Пишется в любом регистре.
  • text - файл содержит текст
  • css - файл содержит CSS.
    • .
Сообщает браузеру, какой MIME-тип данных используется для внешнего документа. Применяется для того, чтобы указать, что подключаемый файл содержит CSS.
meta <meta Атрибуты > Определяет метатеги, которые используются для хранения информации предназначенной для браузеров и поисковых систем. Например, для получения описания сайта, ключевых слов и других данных. Разрешается использовать более чем один метатег. Все они размещаются в head.
<meta charset="кодировка" > charset="кодировка"
    Значения
  • UTF-8
    • - Самая популярная и часто используемая кодировка
  • RU
 Указывает кодировку документа. Кодировка должна быть указана первой в head. Браузер должен узнать о кодировке как можно раньше. Если кодировка указана после других тегов, текст может отображаться неправильно. Кодировка должна быть выставлена и на веб-сервере, и на клиенте.
<meta name="название-метаданных" content="значение"> name="название-метаданных" content="значение" - атрибуты name и content работают в паре, name — указывает, какого типа информация передаётся (например, описание страницы, ключевые слова, автор и так далее), content — содержит саму эту информацию.

Теги meta не должны повторяются с одинаковыми name. На странице должен быть только один тег с name="description", "viewport", "robots" и другие, иначе браузеры и поисковые системы не поймут, какой из них испльзовать. Повторения могут вызвать конфликт или игнорирование тегов.

Описания не должны быть пустыми или слишком длинными. Слишком короткие или, наоборот, перегруженные описания могут ухудшить SEO и просто не будут учтены поисковыми системами. Если описание вообще отсутствует — в таком случае поисковик сам подставит какой-то фрагмент текста со страницы. Это не всегда будет удачно: может попасться неподходящий абзац или фраза без смысла. Если слишком длинное описание: Такой текст просто не влезет в сниппет — его обрежут поисковые системы, и пользователь не увидит главного. Описание должно укладываться в ~150 символов, содержать ключевые фразы (iPhone 15, цена, доставка, цвета), привлекательно выглядеть в поисковой выдаче.

Передает браузерам и поисковым системам дополнительную информацию о странице
name="author" content="Петров" Указывает автора
name="generator" content="WordPress 6.0" Указывает программу или систему, которая использовалась для создания страницы (например, WordPress, Joomla, FrontPage).
name="description" content="Описание страницы для SEO" Краткое описание страницы
name="keywords" content="HTML, метатеги, SEO, веб-разработка" список ключевых слов
name="robots" content="index, follow"
    Значения атрибута content (index, follow стоит по умолчанию):
  • index — разрешить индексацию страницы;
  • noindex — запретить индексацию;
  • follow — разрешить переход по ссылкам на странице;
  • nofollow — запретить переход по ссылкам;
  • none — сокращение для noindex, nofollow.
Управляет индексацией и сканированием.
name="viewport" content="width=device-width, initial-scale=1.0"

Viewport — это область, которую видит пользователь на экране, когда заходит на страницу сайта с любого устройства. Он позволяет указать, какая область просмотра необходима для страницы . viewport необходим для корректной мобильной адаптации и влияет на SEO: поисковые системы отдают приоритет сайтам, оптимизированным для смартфонов. Если этого тега нет или он указан с ошибкой, позиции сайта в выдаче могут пострадать.

    Некоторые основные параметры viewport (content):
  • width определяет ширину области просмотра. Чаще всего используется значение device-width, что означает «использовать ширину экрана текущего устройства».
  • initial-scale задаёт начальный масштаб страницы. Значение initial-scale=1.0 устанавливает масштаб по умолчанию — 100%, без увеличения или уменьшения.
  • maximum-scale и minimum-scale определяют, как сильно пользователь может увеличивать или уменьшать масштаб.
  • user-scalable имеет значения «yes» или «no» и определяет, может ли пользователь масштабировать страницу.
Для адаптивной вёрстки важно, чтобы тег был написан правильно. Даже небольшая ошибка может повлиять на отображение сайта на мобильных устройствах. Если пропущен width=device-width,то браузер не поймёт, что нужно подстроить ширину страницы под ширину экрана смартфона. В результате сайт может отображаться слишком узким или слишком широким, и пользователю придётся вручную уменьшать или увеличивать масштаб. 		Указывает браузеру, как масштабировать и отображать страницу на разных устройствах, особенно на смартфонах и планшетах.
name="theme-color" content="#4285f4"

Цвет задаётся в шестнадцатеричном формате (HEX): #4285F4 — синий; #FF0000 — красный.

Цвет интерфейса браузера.
<meta http-equiv="refresh" content="30"> http-equiv="refresh" content="30". Браузер обновит страницу через 30 секунд. Автоматическое обновление страницы.
<meta http-equiv="refresh" content="5;url=https://example.com/new-page.html"> http-equiv="refresh" content="5;url=https://example.com/new-page.html".

Браузер перейдёт на указанный адрес через пять секунд.

 Настраивает временное перенаправление. Для постоянных редиректов лучше использовать серверные настройки, — так их корректно обработают поисковые системы.
<meta http-equiv="Content-Security-Policy" content="default-src 'self'"> http-equiv="Content-Security-Policy" content="default-src 'self'" Защищает сайт от XSS (content security policy): Этот метатег помогает защитить сайт от XSS-атак, при которых злоумышленник пытается внедрить вредоносный код на сайт. Метатег запрещает загружать ресурсы со сторонних источников, если они не разрешены явно. Обычно такая политика задаётся на стороне сервера, но в простых случаях её можно прописать и через meta — при тестировании сайта или при локальной разработке.
title <title>Название</title> Задает название вкладки. Название вкладки отображается в браузере при просмотре страницы. Размещаются в head.
script <script Атрибуты> Содержание</script> Предназначен для описания скриптов, может содержать ссылку на программу или ее текст на определенном языке. Скрипты могут располагаться во внешнем файле и связываться с любым HTML-документом. Могут располагаться в заголовке или теле HTML-документа в неограниченном количестве.
<script type="тип" src="URL"> Содержание</script> type="тип".
    Атрибут type не чувствителен к регистру и принимает следующие значения:
  • text/javascript - для JavaScript (Значение по умолчанию)
  • text/vbscript - для VBScript.
 Указывает тип MIME для определенного языка (type="тип").
<script type="тип" src="URL"> Содержание</script> src="URL" Импортирует файл со скриптами из внешнего файла. Как только внешние инструкции полностью загружаются, браузер обрабатывает их, словно они были внедрены в текущий документ. Файлы со скриптами JavaScript обычно имеют расширение js и могут размещаться в любом доступном месте.
style <style Атрибуты> селект {свойство: значение; свойство: значение;}</style> Предназначен для определения стилей элементов веб-страницы. Тег style необходимо использовать внутри контейнера head. Можно задавать более чем один тег style.
<style media="all|braille|handheld|print|screen|speech|projection|tty|tv"> селект {свойство: значение; свойство: значение;} </style> Значения all- Все устройства. -braille- Устройства, основанные на системе Брайля, предназначены для слепых людей. handheld - Наладонники, смартфоны, устройства с малой шириной экрана. print - Печатающее устройство вроде принтера. screen - Экран монитора (Значение по умолчанию). speech - Речевые синтезаторы, а также программы для воспроизведения текста вслух. Сюда же входят речевые браузеры. projection - Проектор. tty - Телетайпы, терминалы, портативные устройства с ограниченными возможностями экрана. Для них не должны использоваться пикселы в качестве единиц измерения. tv - Телевизор. Можно устанавливать сразу несколько значений, перечисляя их через запятую. Устанавливает устройство вывода, для которого предназначена таблица стилей. Для каждого устройства — от карманного компьютера до принтера можно определить свой собственный стиль, который бы учитывал специфику устройства и подгонял под него вид веб-страницы.
<style type="text/css"> селект {свойство: значение; свойство: значение;} </style> В качестве значения используется MIME-тип — text/css. Значение по умолчанию - text/css Обязателен в HTML4 и XHTML, не нужен в HTML5. Сообщает браузеру, какой синтаксис использовать, чтобы правильно интерпретировать таблицу стилей. Как правило, браузеры корректно работают со стилями и при отсутствии этого атрибута, он необходим для некоторых старых браузеров, которые могут не распознать содержимое контейнера style.
