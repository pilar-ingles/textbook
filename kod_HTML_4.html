HTML - Теги структуры документа

Тег/Формат Атрибуты Описание
span

<span>Содержание</span>

Пример: в разделе head пишем

<style>.letter {color: red; font-size: 300%;position: relative; top: 7px; </style>}

в основном разделе: <span class="letter">О</span>на нанесла в руках отвратительные

<span style="color: green;">зеленые</span>

id-имя тега

class-использутся, если в разделе head задан стиль

style-задает стиль непосредственно в теге

 Cлужит для строчной вёрстки блоков. Строки (тексты)располагаются последовательно друг за другом. Является универсальным пустым контейнером и определяет отдельные строки, символы или другие строчные элементы для их последующего оформления с помощью CSS стилей, либо для манипулирования ими при помощи скриптов. Если расположить внутри HTML-кода две строки, то они последовательно встанут друг за другом (в строку). Если какая-то из строк дойдет до конца ширины документа, то она будет автоматически перенесена на следующую строку, а разделителем переноса скорее всего послужит пробел. Ширина и высота строки зависит целиком и полностью от её содержимого. Чтобы не задавать каждый раз стиль внутри тега, можно выделить стиль во внешнюю таблицу стилей, а для идентификации тега использовать атрибуты class или id с именем селектора.
div

<div>Содержание</div>

Пример с title <div title="Подсказка, привет!"> Основной текст </div>

Пример с id <div id="box01"> Какой-то контент 1 </div>

Пример с class <div class="classA"> Какой-то контент 2 </div> Пример с множеством class <div class="classA classB classC"> Какой-то контент 3 </div>

Пример с contenteditable <div contenteditable="true">Имя: </div>

<div class="header"> - (можно заменить header)

<div class="main"> - (можно заменить main)

<div class="article"> - (можно заменить article)

<div class="footer"> - (можно заменить footer) и т.д.

title-задает текст всплывающей подсказки, которая отображается при наведении курсора на содержимое контейнера

id-имя тега

class-использутся, если в разделе head задан стиль. Если нужно использовать несколько классов, просто перечислите их через пробел в значении атрибута class

style-задает стиль непосредственно в теге

contenteditable-позволяет пользователю редактировать содержимое элемента прямо на странице. Он принимает значения true (разрешает редактирование) и false (запрещает)

 div (division-("раздел", "секция"))-блочная вёрстка Блоком называется прямоугольник, занимающий полную ширину страницы или её родительского контейнера, и имеющий произвольную высоту (высота элемента зависит от содержимого элемента). Внутри HTML-кода два произвольных блочных тега расположатся на странице друг за другом, последовательно. Каждый из блоков займет полную ширину страницы. Для отображения жестко фиксированные элементы (картинки, формы ввода, видео, таблицы) используется блочный подход. Он используется для разделения содержимого страницы на более мелкие части или разделы. Элементы div часто используются для группировки нескольких абзацев, изображений, заголовков и ссылок. Это позволяет упростить идентификацию и стилизацию различных разделов веб-страницы с помощью CSS. С точки зрения поисковых систем, div является безликим блоком универсального характера. Его рекомендуется использовать для создания структуры страницы. Альтернативно, теги div служат для применения стилей к обширным частям контента или функционируют в качестве родительского элемента для подчинённых компонентов. На более сложных веб-страницах элементы div обычно вложены для отображения отдельных разделов и субрегионов страницы. div может быть вложен в другой div, позволяя создавать многоуровневые структуры для более точного управления контентом. Один из способов стилизации — использование CSS-классов, которые задаются через атрибут class. Когда не надо использовать div ❌ Когда есть семантические теги (header, section, article и т.д.). ❌ Когда div используется «на всякий случай», без реальной нужды. Когда
точно нужен ✅ Когда нужно сгруппировать элементы и применить к ним общие стили или логику поведения. ✅ Когда нужно динамически управлять контейнером, используя JavaScript. ✅ При создании макетов — в связке с CSS Flexbox или CSS Grid Layout. Типичные ошибки: 1-Слишком много вложенных/2-вместо семантических блоков/3-Отсутствие классов и идентификаторов
header

<header>Содержание</header>

 используется вместо <div class="header">

Описывает шапку сайта или страницы, которая находится в верхней части ресурса, и содержит название, логотип,форму поиска, навигационное меню и контактную информацию. Необходима для поисковым систем. Используется для смантической разметки,структурирования контента, размещения общих элементов,описания стилей и вложенных заголовков. Может быть вложен в другие структурные элементы, такие как <article> или <section>, чтобы обозначить заголовок для соответствующего раздела.

main

<mail>Содержание</mail>

используется вместо <div class="main">

Предназначен для размещения основного контента документа. Содержимое тега должно быть уникальным и не дублировать однотипные блоки, которые повторяются в других документах, таких как шапка сайта header, нижний колонтитул footer, меню, форма поиска, информация об авторских правах и т.д. Он не должен быть вложен в теги article, aside, footer, header или nav. В одном документе допускается использование одного тега main. Тег появился в HTML5.

nav

<nav>Содержание</nav>

 используется вместо <div class="nav">

Cоздаут блоки навигации, ссылок, которые ведут на другие веб-страницы, или разделы текущей. В одном HTML документе может содержаться несколько таких тегов (один-для группы ссылок для навигации по сайту, другой - для текущей веб-странице и т.д.). Тем не менее, не все ссылки могут быть помещены внутрь этого тега, а только крупные навигационные блоки. Его нельзя вкладывать такие теги, как footer, address.

aside

<aside>Содержание</aside>

используется вместо <div class="aside">

Тег определяет раздел, в котором содержится информация, дополняющая основной отдел. Эта информация является факультативной, и если ее удалить, основное содержимое не пострадает. Такой информацией могут быть комментарии, справочная информация, списки терминов, коллекция ссылок и т.д. Он не может использоваться для определения текста в скобках. Введен в HTML5.

section

<section>Содержание</section>

используется вместо <div class="section">

Bспользуется для группировки логически связанного контента и создания разделов страницы (блок новостей, контактная информация и т.д.). Может использоваться при создании “посадочных страниц” (landing page) для разделения страницы на логические блоки. Он может быть вложен в тег article, разделяя содержащийся в нем контент на логические блоки. При этом необходимо, чтобы в теге article и вложенном в него теге section были использованы заголовки h1-h6. И наоборот, article может вкладываться в section, если необходимо сформировать тематические блоки в разделе. Теги section могут вкладываться друг в друга, но при этом иерархия заголовков не должна нарушаться. Тег нельзя использовать в в качестве универсального контейнера для создания структуры страницы. Это нужно делать с использованием div. Используется начиная с HTML5.

article

<article>Содержание</article>

используется вместо <div class="article">

Bспользуется для обозначения раздела страницы, логического блока, в котором содержится независимая, завершенная композиция с собственным смыслом, например запись в блоге, сообщение на форуме, статья в газете/журнале, пользовательский комментарий и т.п. В одном документе может быть неограниченное количество тегов article. Введен в HTML5.

footer

<footer>Содержание</footer>

используется вместо <div class="footer">

Формирует нижний колонтитул (“подвал”) веб-сайта или раздела. В нем размещается информация об авторских правах, контактные данные, ссылки для навигации и т.д. В пределах одной страницы можно разместить несколько тегов footer, например, можно разместить нижний колонтитул внутри тега article для хранения относящейся к статье информации (ссылки, сноски, и т.д.). В нем могут содержаться другие HTML элементы, кроме footer и header. Если в нижнем колонтитуле содержится контактная информация, то ее лучше размещать в теге address. Введен в HTML5.

adress

<adress>Содержание</adress>

используется вместо <div class="adress">

Используется для выделения контактной информации владельца сайта либо автора статьи. В качестве контактной информации для обратной связи может быть указан адрес, телефон, e-mail, ссылка на сайт и т.д. Часто браузеры добавляют перед элементом и после него разрыв строки, а информацию в теге отображают курсивным начертанием. Если информация относится ко всему документу, то тег помещают внутрь тега body. Контактной информации автора статьи помещается в тег address в элементе article.
